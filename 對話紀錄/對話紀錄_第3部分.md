可以,但「測光強度」有 3 種做法,精準度和工程量差很多:

## 三種選項

| 方法 | 拿到什麼 | 在 Expo 怎麼做 | 缺點 |
|------|---------|----------------|------|
| **A. 環境光感測器**(lux) | 直接的勒克斯數值 | 需第三方 native module(`expo-sensors` 沒包含 light sensor),要寫 config plugin | 不是每台 Android 都有;校正不一致;**只測手機上方光,不一定等於燒杯處光** |
| **B. 讀 EXIF / 相機參數** | ISO、快門、光圈 → 算 EV(曝光值) | `react-native-vision-camera` 拍完直接拿得到 | 如果我們**鎖死 ISO/曝光**(Phase 1 計畫),EV 會永遠一樣 → 失去意義 |
| **C. 灰卡 / 白卡校正** | 影像中參考色塊的 RGB | 拍照時把灰卡放進畫面,從像素讀取 | 要多一個物件(便宜、可印一張);但**這才是專業做法** |

## 我的建議 — 用 C,丟掉 A 和 B

因為:
1. **A 量的不對** — 手機光感測器在螢幕上方,你燒杯放桌面,真正打到藻類的光跟手機讀到的不一樣
2. **B 自打嘴巴** — 我們 Phase 1 就是要鎖相機參數,鎖了就拿不到光強資訊
3. **C 一勞永逸** — 燒杯旁邊放一張**灰卡(18% 灰)或白卡**,每次拍照都入鏡。色彩分析時用灰卡的 RGB 當基準,**不管環境光怎麼變,藻類顏色除以灰卡顏色 = 真實顏色**

灰卡網路上一張幾十塊,或印一張色卡也行。

## 折衷 — A + C 都做

如果你想**順便記錄**現場光強(實驗筆記用),A 加進來也不貴(一個套件、一個讀數)。但**色彩校正主力還是 C**。

要怎麼走?
- **C only**(最簡單,Phase 1 加一個「請把灰卡放入畫面」的 overlay)
- **A + C**(加一個 lux 顯示在畫面角落,純參考用)
- **暫時不處理**,Phase 1 先鎖設定就好,校正後面補

| 加 `npm install xxx-native-module` | 重 build APK + 重發 |
| 改 `app.json` plugins / permissions | 重 build APK + 重發 |
| 升 SDK 版本 | 重 build APK + 重發 |

## 綠框 ≠ 要綠色

綠框只是**畫面上的引導線**(UI overlay),拍照時**不會印到照片裡**。它的顏色(綠)只是為了跟白卡黃框區分開來。

## 為什麼會影響 — 因為是「平均」

App 的演算法很簡單:**把框內所有 pixel 的 RGB 平均**。

```
框內混了 50% 燒杯 (深綠) + 50% 木桌 (淡黃灰)
↓
平均出來是「淡綠灰」 — 不是燒杯真實顏色,也不是桌面真實顏色
```

**影響有多大**:
- 綠框包含的範圍有 30% 是桌子 → 燒杯 RGB 偏離 30%
- 黃框只有一半是白卡 → 白卡基準失準 → **色比全錯**

|---|---|
| **綠框(燒杯)** | 框內**全部**要是燒杯+藻液(別讓桌面、手指、瓶蓋進來)|
| **黃框(白卡)** | 框內**全部**要是白卡 / 白紙(別有桌面、邊緣、陰影)|

## 為什麼

理論上白卡校正能消除光的影響(只要光是線性的)。但**現實中 JPEG 不線性**:
- 相機輸出走過 **gamma + tone mapping**(S 曲線)→ pixel 值跟實際亮度不成正比
- **暗部壓得更扁**,亮部反應更平
- 低光時 G 通道掉最多(感光最弱)

具體看數值:
- 白卡掉 23%(183→143)
- 燒杯掉 60%(114→45)
- 比例不一樣 → 校正失敗

## 但!**色比的「形狀」還在**

把每組除以最大通道:
- 開燈:(0.97, 0.63, **1.00**)— B 最高,R 接近,G 偏低
- 關燈:(0.95, 0.49, **1.00**)— 同樣模式 — B 最高,R 接近,G 偏低

**通道間的相對關係保留**(R/B 還在 0.95~0.97),但**絕對亮度改變了**。

## 實務建議

**不要做極端測試**,實際使用的光照範圍應該:

| 場景 | 白卡 RGB 範圍 | 校正可不可信 |
|------|-------------|-------------|
| 完全暗(剛剛) | < 150 | ❌ 失效 |
| 一般室內 | 150–230 | ✅ 可信 |
| 室外 / 強光 | 230+ 但 < 250 | ✅ 可信 |
| **過曝**(白卡飽和) | 250+ 個別通道 = 255 | ❌ 失效 |

**正常拍攝**(白卡在 150–230 之間)色比應該都很穩。剛剛差太多是因為**白卡掉到 141** 已經接近暗部非線性區。
